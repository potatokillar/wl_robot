# 文档编写规范 (Style Guide)

为了保证文档的质量和一致性，所有新增文档必须遵循以下规范。

## 1. 文件命名
- 采用 **小写英文** 命名，单词间使用连字符 `-` 分隔。
- 示例：`api-reference.md`, `getting-started.md`。

## 2. 标题层级
- 一级标题 (`#`) 仅用于文档标题，每页只能有一个。
- 二级标题 (`##`) 用于主要章节划分。
- 三级标题 (`###`) 用于子章节。
- 避免使用四级以上标题，如果层级过深，请考虑拆分文档。

## 3. 代码块
- 必须指定语言类型以启用语法高亮。
- 示例：
  ```cpp
  // Good
  void Init() {
      // ...
  }
  ```

## 4. 图表
- 优先使用 Mermaid 语法绘制流程图和时序图，以便于版本控制和维护。
- 如果必须使用图片，请存放在 `assets/images` 目录下，并使用相对路径引用。

## 5. 术语表
- **Quadruped**: 四足机器人
- **Arm**: 机械臂
- **HAL**: Hardware Abstraction Layer (硬件抽象层)
- **Node**: 这里的 Node 指代 ROS 风格的业务节点（如 `UserQuadNode`），而非物理节点。
